DOM MINIPULATION SPEED TESTS

Here we are populating a select list with 300 select options using 3 different methods

METHOD 1: This is the slowest method of the 3. It is unsing array map to add innerHtml on every itteration. This is a no go because you are minipulating the dom on every loop. This uses alot of client system pefromce. This is a small test but takes 230ms +- to complete. Anything more than 500 and the browser tends to freeze.

METHOD 2: One of 2 methods used to minipulate the dome in under 2ms. What we are doing here is building up a string and then adding it as innerHtml in 1 call to the document element. This is far more efficient as it requires 1 call to minipulate the dom element.

METHOD 3: Almost identical to the first one in that we minipulate the dom once, however we are building up a document fragment. Because we are not interfacing with the document and because its not part of an document tree it incurrs 0 peformance impact on the document.
